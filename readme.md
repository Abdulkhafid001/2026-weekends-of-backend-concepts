- Idempotent flow
. use a unique autogenerated key to track transactions
. when a transaction is processed once, do not process again
. always check if transaction is new before creating  
. get a unique key from the frontend, and process the transaction, 
save the transaction to the DB, anytime we process a new transaction we check the
key and a user session so we don't charge a user twice when he/she clicks the pay button twice
- Django view checks if that key already exists in cache/DB before processing — if it does, it returns a duplicate response instead of charging again
. After a successful payment, the button is locked so the same key can't be reused from the frontend either
